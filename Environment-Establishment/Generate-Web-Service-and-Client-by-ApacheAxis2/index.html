開發環境

  • Java SE Development Kit 6u45
  • Apache Tomcat 6 (6.0.48)
  • Apache Axis 2 (1.7.1)
    要下載bin及war檔
  • Eclipse IDE for Java EE Developers (4.6)
  • MS SQL Server 2016

相關軟體位置

  • Tomcat 6 :
    D:\apache-tomcat-6.0.45
  • Axis2 :
    bin檔案放在D槽並改為axis2 (D:\axis2)
    將axis2.war檔放在D:\apache-tomcat-6.0.48\webapps，便會自動生成axis2資料夾。
  • Workspace :
    C:\Users\adl\workspace\WS2MSClient

安裝 SQL Server JDBC Driver

Step 1 下載

要使用 Java 連線到 SQL Server 必須要使用適當的 JDBC driver，在微軟的 MSDN 網站有提供免費下載。
Microsoft JDBC Driver 6.0 for SQL Server

Step 2 安裝

解壓縮 sqljdbc_6.0_cht 可得到四個 jar 檔，Java Runtime Environment (JRE) 的版本支援及相關 JDBC API 取決於下列 JAR 檔案：

  • Sqljdbc.jar 需要 JR E 5 版並支援 JDBC 3.0 API
  • Sqljdbc4.jar 需要 JRE 6 版並支援 JDBC 4.0 API
  • Sqljdbc41.jar 需要 JRE 7 版並支援 JDBC 4.1 API
  • Sqljdbc42.jar 需要 JRE 8 版並支援 JDBC 4.2 API

因為我們是 JRE6，因此將 sqljdbc4.jar 複製到 tomcat 目錄下的 lib，以及 D:\apache-tomcat-6.0.48\webapps\axis2\WEB-INF\lib 中。


環境設定

安裝好 Java 和 Tomcat 之後，要先修改環境變數，在本機上按右鍵，選擇 內容 -> 進階系統設定 -> 進階 -> 環境變數


點選新增系統變數

加入系統變數 JAVA_HOME，位置是 Java jdk 的位置

加入系統變數 CLASSPATH，位置是 Java jdk\lib 的位置

在系統變數 Path最後面加上 java bin 的位置

加入系統變數 AXIS2_HOME，位置是 D:\axis2 的位置

資料庫連線參數

開啟 D:\apache-tomcat-6.0.45\webapps\axis2\WEB-INF\conf\axis2.xml，將連線到資料庫的相關參數寫在裡面；
這邊會用到四個參數

建立 Eclipse 專案

Step1 在 Eclipse 建立一個新的 Java project

左上角的 File →New→ Java Project ，在 Project name 輸入 WS2MSSQL，按 Next

選 Libraries 分頁按 Add Library

選 User Library 後按 Next

新增一個User Library 叫做 Axis2 Libraries

新增完畢後，案右邊的 Add External JARs，新增 Axis2 的 jar 檔，jar 檔位置是在 D:\axis2\lib

新增完畢後，要將 Axis2 Libraries 打勾才算有使用，後按 Finish 完成

 

Step 2 建立資料庫連線

首先建立新的 Java Class ，命名為 DBtestServiceLifeCycle，並實作 org.apache.axis2.engine.ServiceLifeCycle 介面；必須實做兩個方法

  • public void startUp(ConfigurationContext configctx, AxisService service)
    • this will be called during the deployement time of the service. irrespective of the service scope this method will be called
  • public void shutDown(ConfigurationContext configctx, AxisService service)
    • this will be called during the system shut down time. irrespective of the service scope this method will be called

Step 4 建立 Service

建立新的 Java Class ，命名為 DeptDBService，我們來實做兩個 method

  2. 無傳入參數，傳回 java bean
    撰寫 public Dept[] listAllDept() method，這是要給 Client 端呼叫的方法

打包 JAR 檔

在 Project上 按右鍵，選 Export…

選擇 Java →JAR file ，按 Next

將剛剛寫好的檔案都打勾起來，下面打上儲存的路徑，然後按 finish

佈署 service

用瀏覽器開啟 Axis2 管理頁面，選左邊的 Upload Service (http://localhost:8080/axis2/)

按選擇檔案，選擇剛剛包好的檔案，之後按開啟
按下 Upload

按左邊的 Available Services 可以看到剛剛佈署完成的 DBSampleService

可以看到我們剛剛寫的兩個 methods ( listAllDept、QueryNameById )
按下 DBSampleService 即可看到 WSDL 定義

撰寫 Client 測試

Step 1 建立 client 專案

先建立一個一般的 Java project 名為 WS2MSClient，同樣的完成後匯入 Axis2 的 Library 。
接著要替你剛剛寫的 Web Services 產生 Client 端的程式碼:

  1. 開啟 CMD 進入至 Axis2 的 bin 目錄下 ex: D:\axis2\bin
  2. 執行 D:\axis2\bin> wsdl2java.bat -uri http://localhost:8080/axis2/services/DBSampleService?wsdl -o C:\Users\adl\workspace\WS2MSClient

PS:-uri WSDL 的 uri 位置
-o 輸出 stub classes 的位置. 預設會增加 src 目錄.
-p 設定輸出 stub classes 使用的 package.
